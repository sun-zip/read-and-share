## 💠 목표

- 백엔드 시스템에서 발생하는 에러/버그의 탐지 및 해결
    - 백엔드 시스템에서 발생하는 에러와 버그를 탐지하고 해결하여 안정성을 높임
    - 예상치 못한 오류 발생 시 원인을 빠르게 분석하고 대응
- 팀 협업을 통한 문제 해결 능력 강화
    - 팀 단위 협업을 통해 실무에서 요구되는 소통 및 협업 기술을 익힘
    - Git Flow 전략을 적용하여 효율적인 코드 관리 및 버전 컨트롤
- 테스트 코드를 작성하여 구현과정에서 발생할 에러를 미리 탐지하고 해결
    - JUnit5 및 Mockito를 활용한 단위 테스트 및 통합 테스트 진행
    - 테스트 자동화를 통해 코드의 신뢰성을 높이고 유지보수를 용이하게 함
- API 문서화 및 관리
    - Swagger 또는 Postman을 활용하여 API의 가독성을 높이고 유지보수를 편리하게 함
- 성능 개선 및 최적화(고려)
    - JMeter를 활용한 부하 테스트를 수행하여 병목 구간을 분석하고 최적화 진행
    - Prometheus 및 Grafana를 활용하여 시스템 성능을 모니터링하고 개선 방향을 도출

### 깃 커밋 메시지 컨벤션
태그: 제목의 형태이며, : 뒤에만 space가 있다.
|태그|용도|
| --- | --- |
| feat | 새로운 기능 구현 |
| add | FEAT 이외의 부수적인 코드 추가, 라이브러리 추가, 새로운 파일 생성 |
| hotfix | issue나 QA에서 급한 버그 수정에 사용 |
| fix | 버그, 오류 해결 |
| del | 쓸모 없는 코드 삭제 |
| docs | README나 WIKI 등의 문서 개정 |
| style | 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우 |
| refactor | 코드 리팩토링 |
| test | 테스트 코드, 리팩토링 테스트 코드 추가 |
| chore | 자잘한 수정이나, 빌드 업데이트 |
</details>
